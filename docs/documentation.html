Getting started

Freally Teleprompt is a standalone, cross-platform teleprompter for creators, streamers, podcasters, rappers, singers, presenters, and actors. It runs on Windows, macOS, and Linux, opens as a single window — script library, editor, preview, and projector controls — and needs no account. One character-based scroll engine drives the preview, the talent projector, and the LAN mirror together, so they never drift, even at different font sizes.

Status: active development. Freally Teleprompt ships along a phased roadmap — 0.100.0 (scaffold + the ported char-based engine and docs site), 0.200.0 (the standalone teleprompter core: script library, caesura chips, projector, LAN mirror, transport, chars/sec & BPM, appearance, read-aloud) and 0.300.0 (offline dictionary autocomplete) are done; 1.0.0 (polish, signing, and release) is next. Download builds from the Releases page.

Install & first run

  1. Download your platform's installer from the Releases page — Windows NSIS .exe / .msi, macOS universal .dmg, or Linux AppImage / .deb — then install and launch it.
  2. On first launch, the EULA gate appears: scroll to the end, then click I Agree (or Decline & Quit). The app is unusable until you accept, and acceptance is remembered — saving Settings never wipes it. If the EULA version changes in a later release you'll be asked once more.
  3. The window opens to the script library and editor. There's no login and nothing to activate — every core feature is available immediately.
  4. Type or paste a script (see Write a script), set your pace, and press play to watch it scroll.

Write a script

Your scripts live as real files you can keep, reopen, and back up. Write directly in the editor, or paste from anywhere — the editor cleans up pause markers as they land.

The script library

  1. Scripts are saved as .ftscript files. From the library you can create, open, save, rename, and delete them.
  2. A recents list gets you straight back to what you were last working on, and autosave keeps your edits so a script isn't lost if you forget to save.

The editor

  1. Type your script directly in the editor, or paste text from a document, an email, or the web.
  2. Insert timed pauses with the inline -- caesura marker — it becomes an atomic pause chip (see Pause chips). Pasted text is normalized so any loose -- lands as a clean chip.
  3. Save to a .ftscript file when you're happy; autosave protects work in progress in the meantime. The same script then drives the preview, the projector, and the LAN mirror.

Pause chips (caesuras)

A caesura is a timed pause written right into the script as -- . Freally Teleprompt renders each one as an atomic chip and lights them one at a time as the scroll reaches them, so the reader always knows exactly where to breathe.

  1. Type -- (space, two dashes, space) anywhere in a line to drop a caesura — a short timed pause. For a longer hold, use --N , where N is the number of seconds; the char-based engine holds the scroll for that long when it reaches the marker.
  2. The chip is atomic: the caret can only sit before or after it, never inside; a single Backspace or Delete removes the whole marker; and Shift+←/→ selects the entire chip at once.
  3. Copy and paste carry the chip intact, and paste-normalization turns any of --, --, or -- into a clean -- so markers stay consistent no matter where the text came from.
  4. During playback each caesura is lit one at a time and the scroll pauses for its duration, then resumes — the same on the preview, the projector, and the LAN mirror.
Tip: A bare -- is treated only as a pause — during Read aloud it becomes a short spoken pause and is never read out as the word "dash".

Pacing: chars/sec & BPM

Speed here is a real reading pace, not an arbitrary 1–10 dial. Pace the scroll in characters per second, or switch to BPM to read lyrics and verses in time with music. Both modes drive the same character-based engine.

Characters per second

  1. Set the pace in characters per second — a genuine reading rate. Because it's measured in characters, the preview, projector, and LAN mirror all advance at the identical pace even when they use different fonts or window sizes.

BPM mode

  1. Toggle to BPM mode to keep time with a beat — ideal for rap, R&B, and sung material. BPM is clamped to a musical range of 20–250.
  2. Switch back to chars/sec at any time; the toggle just changes how the same engine is clocked.
Live re-pacing: changing either the chars/sec value or the BPM re-paces the running scroll immediately — no need to stop and restart.

Playback & transport

Drive the scroll with real transport buttons, the mouse wheel, or the keyboard. The current word stays in view the whole time.

  1. Use the on-screen SVG transport buttons — play, pause, stop, rewind, and fast-forward — with hold-to-repeat on rewind/fast-forward for quick scrubbing.
  2. Seek anywhere: spin the mouse wheel to scrub through the script, or click a word to jump the scroll (and start) at that point.
  3. Drive the same transport from the keyboard shortcuts when your hands are off the mouse.
  4. A reading-guide line pins the current word near the top of the view, and the last-lit word never scrolls out of sight, so the reader always has the next line in front of them.

Start countdown

An optional pre-roll so the reader gets a beat before the text starts moving.

  1. Turn on countdown before start in Appearance. When it's armed and you press play, a countdown runs first, then the scroll begins — giving the talent a moment to settle before the first word moves.
  2. Leave it off for an instant start; the setting is remembered with the rest of your appearance preferences.

Talent projector

Put the script on a second screen for the talent — full-screen, optionally mirror-flipped for beam-splitter glass, and kept in step with a separate device over the LAN. One char-based engine means the projector and your preview always agree.

Open the projector

  1. Open the projector window and target the external display the talent is reading from. The preview and projector lay the script on the same fixed-width stage and scale to fit, so they wrap at the identical column and the projector's layout doesn't reflow when the window resizes.

Mirror mode (beam-splitter glass)

  1. Turn on mirror-flip so the text reads correctly through the angled glass of a beam-splitter teleprompter rig, where the reflection would otherwise reverse it.

LAN mirror

  1. Use the LAN mirror to show the same scroll on a physically separate device (a phone or tablet on your network). It follows the shared character offset, so the remote view is on the same word as the operator — there's no arbitrary "scroll speed" to reconcile.
  2. Turn it on in Settings → Mirror to my network. It is off by default, and "allow other devices" is a second, separate switch — with it off, the mirror answers only on this computer. Apply, and a link and QR code appear; scan or open the link on any device on the same network. The join page walks through it.
  3. Each device sets its own text size on the mirror page without changing what anyone else sees.
  4. The mirror page is served by the app itself, not by this website — a browser will not let an HTTPS page talk to a plain-HTTP device on your own network, and this way your script never travels through anyone's server.
The mirror is read-only, and keyed. Nothing a browser sends can start, stop, seek, or edit anything — there is no write path at all. The link carries a key that is regenerated every time the app launches. Because it is plain HTTP on your own network, the key is not encrypted in transit: use the mirror on a network you trust, not on public Wi-Fi.
Beam-splitter check: whether mirror-flip reads correctly through real glass, and whether the LAN mirror holds sync on a separate device, can only be confirmed on physical hardware — verify both on your actual rig before a shoot.

Read aloud

Preview your script spoken aloud through your operating system's own speech engine, with the highlight following along. Read aloud is a preview-only mode — it never appears on the projector and never touches the shared scroll state.

  1. Tick "Read aloud with per-OS speech synthesis". The script is spoken through your OS's built-in voices — Windows OneCore/SAPI, macOS AVSpeechSynthesis, and Linux via the system Speech Dispatcher (spd-say) or espeak-ng. No speech engine is bundled; it uses the voices you already have installed.
  2. While reading, the modal's transport (play / pause / stop) and seek / click / drag drive the speech, and the highlight follows the spoken word in the preview. Changing the speed re-rates the voice live.
  3. The checkbox is disabled while a read is engaged (playing or paused) and re-enables only after Stop, so it can't be flipped on or off mid-speech.
  4. Where the engine reports word boundaries (Windows / Chromium) the highlight locks to the exact spoken word; on macOS / Linux it falls back to a pace-matched highlight at your chars/sec so it still moves in step. Caesuras and line breaks become short spoken pauses, and a bare -- is never said as "dash".
Note: Read aloud produces real audio through your OS voices, so confirm it actually speaks (and pauses at your caesuras) on the machine you'll present from — especially on Linux, where it shells out to the speech daemon you have installed.

Autocomplete

As you write, Freally Teleprompt offers dimmed ghost-text suggestions from a per-language dictionary. It's instant, offline, and on by default — zero-config, needing no key and no download.

  1. Keep typing in the editor; relevant word and phrase suggestions appear inline, dimmed, within a single keystroke. Finish a word and press space and it will offer to complete a whole common phrase.
  2. Press Tab to accept a suggestion — it commits and turns to normal text color. Press Esc to dismiss it, or simply keep typing to ignore it.
  3. Suggestions are per-language. Each of the 18 languages ships its own word and phrase tables — about 464,000 words and 345,000 phrases in total — and only the language you're writing in is loaded.
  4. Turn it on or off, and choose which language it suggests from, under Settings → Editor. The suggestion language is separate from the app's language, so you can run the app in English and write your script in Polish.
Where the words come from: the tables are built from the Tatoeba sentence corpus (CC BY 2.0 FR) and ship inside the installer. The NOTICE file included with the app records the full attribution.
Why a dictionary, not AI: for autocomplete a dictionary is the right tool — instant (well under a millisecond), deterministic, predictable, and entirely offline. It can't invent something you didn't ask for, and it needs no model, no download, and no network at all.

Voice control

Run the prompter hands-free by voice. You train a small set of commands in your own voice, and the app matches what you say against your own recordings — on this device, with no model and no network. It is off by default.

  1. Open Settings → Voice and tick "Control the prompter with my voice." Choose when it listens: only while you hold a button (push-to-talk) or always, while enabled.
  2. Under Your commands, press Record next to a command — Play, Pause, Faster, Slower, Next pause, or Back to top — and say your word. Record each one two or three times; more takes make it steadier. Any word can stand for any command.
  3. On the prompter, a red ● Listening indicator shows whenever the microphone is open. In push-to-talk mode, hold Hold to talk and speak; in always mode it listens continuously while enabled. Say a trained word and the prompter obeys.
  4. A word it does not clearly recognise is ignored, never guessed — a prompter that jumped on a mishear would be worse than one that waited.
Note: voice commands need a microphone, and recognition is speaker-dependent — train your commands on the machine and in the voice you will present with. Test them before you go live.
Private by construction: commands are matched by classic signal processing against your own recordings — no model, no training data, no network. The microphone is open only while listening, audio is processed in memory, and nothing you say is ever written to a file.

Coming — follow my reading. A separate Follow my reading option scrolls the script to keep up as you read it aloud, recognising the script's own words on your device. It steps aside to manual scrolling the moment it loses your place, so it never scrolls somewhere wrong, and it drives only the operator preview — never the projector. It is opt-in and needs a speech model installed; where the model isn't present the option is greyed out with a note.

Appearance, languages & settings

Tune how the prompter looks, pick your language, and manage settings safely — nothing changes until you apply it.

Appearance

  1. Set the font family, size, weight, and color, plus margins, line height, and the reading-guide position so the current line sits where the talent looks.
  2. Enable the optional countdown before start here too.

Settings & language

  1. Apply means apply. Settings edit a private draft — nothing takes effect until you click Apply; cancelling restores exactly what you had.
  2. Language: choose from 18 languages, or leave it on Same as my system; the interface updates immediately, and the autocomplete dictionary follows your choice.
  3. Theme: dark or light, applied instantly across the whole interface. (The reading surface itself stays high-contrast for the talent whichever theme you pick.)
  4. Later polish adds an onboarding tour and an accessibility pass as the app approaches 1.0.0.

Problems & updates

If something goes wrong, Freally Teleprompt helps you report it — but never behind your back. Nothing is transmitted without your click, and you read the exact text first.

  1. If the app ever stops unexpectedly, it writes a report to a local file and offers to restart. When it comes back, the report opens automatically with the crash details attached.
  2. The dialog shows exactly what will be sent in a read-only field. Your home folder path and username are redacted from it automatically.
  3. Send it however you prefer: open a GitHub issue, compose in Gmail, or use your own mail app — each opens pre-filled. You press send, not us. There is no server and no account.
  4. You can also open the reporter any time from the toolbar to describe a problem that didn't crash anything.
  5. Updates: once per launch the app asks GitHub whether a newer release exists. If one does, you get a single prompt showing the version number and the real release notes, with an explicit yes or no. Offline, or already current, it stays silent — it never nags.
What the update check sends: nothing about you. It fetches one small file from the public releases page; nothing downloads or installs until you answer the prompt.

Privacy

Freally Teleprompt is local-first and account-free. Here is exactly what it does — and does not — do with your data.

  1. No account, no cloud, no telemetry. The app never requires a login and doesn't phone home. Your scripts stay on your device as .ftscript files where you save them.
  2. No AI, anywhere. There is no model in the app, none to download, and no API key to paste. Nothing you type is sent to a provider, because there is no provider.
  3. Autocomplete is fully offline. The per-language dictionary tables ship inside the installer and run entirely on your machine — no key, no download, no network.
  4. The projector and LAN mirror stay on your network. They target your own displays and devices; nothing is served to the internet.
  5. The only outbound request is the update check, which asks GitHub whether a newer release exists. It sends no script content.
